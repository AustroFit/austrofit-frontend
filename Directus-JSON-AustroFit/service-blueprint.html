Tech-Architektur (festgelegt)

  • PWA (SvelteKit) als Basis – kein App-Store in Phase 1, direkter URL-Zugang ermöglicht sofortiges Testen ohne Store-Review.
  • Capacitor wird gezielt nur als Bridge für Health Connect (Android) und HealthKit (iOS) eingesetzt – alle anderen Features bleiben reine PWA.
  • Punkte-Engine als SvelteKit API Routes + Direktus REST; Onboarding-Bonus via /api/auth/register gebucht (kein Direktus Flow nötig).
  • E-Mail + Passwort als Registrierungs- und Login-Methode (Direktus Magic Link in verfügbarer Konfiguration nicht nutzbar – bewusste Entscheidung).

Onboarding-Flow (festgelegt)

  • Einstieg via Gesundheitswegweiser (SEO-Artikel + Unauth-Quiz) oder direkt via Landingpage / Social Ad.
  • Conversion-Trigger: Nach Quiz-Abschluss Hinweis 'Du hättest X Punkte verdient – jetzt registrieren.'
  • Registrierung: Benutzername + E-Mail + Passwort (/registrierung) → Login → direkt ins Dashboard (kein Setup-Wizard). Login-Seite verlinkt korrekt auf /registrierung.
  • 20P Onboarding-Bonus wird server-seitig in /api/auth/register gebucht (idempotent). Health-Permission contextual beim ersten Tracking-Versuch. PWA Install-Prompt erst nach erstem Dashboard-Besuch.

Kritische MVP-Hypothesen

  • Engagement: ≥50% der Erstnutzer lösen ≥3 Challenges in 14 Tagen.
  • Belohnung: ≥30% der aktiven Nutzer lösen ≥1 Gutschein/Monat ein.
  • Partner: ≥70% der gelisteten Angebote werden im Q1 mindestens einmal eingelöst.
  • Fraud: Schritt-Spike-Filter (>20.000 Steps/h) als Minimalschutz Phase 1.

Phase 2 Prioritäten

  • Gamification: Challenges, Gewinnspiele, Maskottchen, Leveling System live schalten.
  • Community: Events, Bestenliste, Invite-a-Friend Referral-Programm.
  • Fraud Phase 2: ML-Anomalie-Scoring (Isolation-Forest, Schwellenwert 0,85).
  • Stripe + lexoffice für Partner-Paket-Abrechnung (Plus/Premium/Nachhaltigkeit).
